Configuration¶

BOOST_STATIC_VIEWS_USE_BOOST¶

If defined, Boost.Config will be used to detect compiler-specific functionality.

Specifying the error handling policy¶

The following macros allow one to configure how assertion failures are handled. The default policy is BOOST_STATIC_VIEWS_THROW_ON_FAILURES.

BOOST_STATIC_VIEWS_DISABLE_CHECKS¶

If defined, assertions are replaced by calls to __builtin_assume (if the compiler supports it). Use this macro to achieve the best performance.

BOOST_STATIC_VIEWS_THROW_ON_FAILURES¶

If defined, an exception of type assertion_failure will be thrown in case of an assertion failure. Use this macro during development because assertion_failure::what() provides some useful information about the location of the failure.

BOOST_STATIC_VIEWS_TERMINATE_ON_FAILURES¶

If defined, std::terminate will be called on assertion failures. Use this macro in environments that do not support exceptions.
